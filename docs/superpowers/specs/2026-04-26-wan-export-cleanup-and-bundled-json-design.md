# Wan Export Cleanup And Bundled JSON Design

## Goal

After a refined sample successfully produces Wan export targets, copy the final 4D video into each exported target directory, aggregate per-frame Wan pose metadata and raw SMPL outputs into single JSON files per target, and clean up the heavy SAM4D intermediate files while preserving lightweight runtime/debug artifacts for tracing.

## Scope

This change must:

- copy the final rendered 4D mp4 into each exported Wan target directory
- write one aggregated `pose_meta_sequence.json` file per exported target
- write one aggregated `smpl_sequence.json` file per exported target
- clean the refined sample working directory after successful Wan export
- preserve `sample_runtime.json` and `debug_metrics/` in the sample working directory
- keep the current external Wan summary, UUID mapping, skipped-target report, and issue ledger behavior intact

This change must not:

- delete sample-level debug/runtime artifacts needed for postmortem inspection
- delete the sample working directory itself
- run cleanup when no Wan target directories were exported
- replace existing per-target media outputs such as `target.mp4`, `src_pose.mp4`, `src_face.mp4`, `src_bg.mp4`, `src_mask.mp4`, or `src_ref.png`

## Current State

The refined offline pipeline writes a sample working directory under `runtime.output_dir`. That directory can contain heavy intermediate data such as:

- `images/`
- `masks/`
- `masks_raw/`
- `masks_refined/`
- `rendered_frames/`
- `rendered_frames_individual/`
- `mesh_4d_individual/`
- `focal_4d_individual/`
- `completion_refined/`
- final `4d_*.mp4`

Wan export writes per-target outputs either into the sample-local fallback directory or into the configured external `wan_export.output_dir`. It currently writes per-frame Wan pose JSON only when `save_pose_meta_json` is enabled.

`pose_meta_json` is not raw SMPL. It is Wan-specific normalized 2D pose metadata generated by `build_wan_pose_meta()`, including:

- body keypoints
- left hand keypoints
- right hand keypoints
- face keypoints
- image/frame identifiers and canvas size

There is no bundled raw SMPL JSON today.

## Design

### Success Condition

The cleanup and copy steps only run when:

- the refined sample completes successfully
- Wan export is enabled
- Wan export produced at least one target directory

If no Wan targets were exported, keep the sample working directory intact for debugging.

### New Wan Export Config Fields

Add these fields under `wan_export`:

- `copy_rendered_4d_to_targets`
  - `bool`
  - default `True`
  - copy the final `4d_*.mp4` into each exported target directory as `4d.mp4`

- `cleanup_sample_workdir_after_export`
  - `bool`
  - default `True`
  - remove heavy refined intermediates after successful Wan export

These fields are orthogonal to the earlier face-presence skip config and only affect post-export packaging/cleanup.

### Per-Target Bundled JSON Files

Each exported target directory `<sample_uuid>_targetN` will gain:

- `pose_meta_sequence.json`
- `smpl_sequence.json`
- `4d.mp4`

#### `pose_meta_sequence.json`

This is a single aggregated file containing all Wan pose metadata records for that target in frame order.

It is derived from the same payloads currently used for:

- `src_pose.mp4`
- optional per-frame `pose_meta_json/*.json`

The file must contain:

- `sample_id`
- `sample_uuid`
- `track_id`
- `source_path`
- `frame_count`
- `records`

`records` is an ordered list of the existing per-frame Wan pose metadata objects.

#### `smpl_sequence.json`

This is a single aggregated file containing raw per-frame `person_output` records for that target in frame order.

It exists specifically to support debugging and comparison against:

- `pose_meta_sequence.json`
- `src_pose.mp4`
- the final `4d.mp4`

The file must contain:

- `sample_id`
- `sample_uuid`
- `track_id`
- `source_path`
- `frame_count`
- `records`

`records` is an ordered list where each entry contains:

- `frame_stem`
- `person_output`

`person_output` must preserve the original fields from refined export, with NumPy arrays converted to JSON-safe lists.

### Per-Frame Pose JSON Compatibility

If `save_pose_meta_json` is enabled, keep the existing per-frame `pose_meta_json/` directory.

The new aggregated `pose_meta_sequence.json` is additive and exists even when `save_pose_meta_json` is disabled.

This keeps backward compatibility while giving a single-file debugging artifact.

### 4D Video Copy

After refined 4D generation completes and Wan export finalization returns exported target directories:

- locate the final sample-level `4d_*.mp4`
- copy it into each exported target directory as `4d.mp4`

The copied filename is normalized to `4d.mp4` to avoid exposing sample-local timestamp naming in the external target package.

The sample-local `4d_*.mp4` remains in place until cleanup runs.

### Cleanup Policy

When `cleanup_sample_workdir_after_export` is `True`, delete these sample working-directory paths if they exist:

- `images/`
- `masks/`
- `masks_raw/`
- `masks_refined/`
- `rendered_frames/`
- `rendered_frames_individual/`
- `mesh_4d_individual/`
- `focal_4d_individual/`
- `completion_refined/`
- sample-local fallback `wan_export/`
- sample-local `4d_*.mp4`

Preserve:

- `sample_runtime.json`
- `debug_metrics/`

This preserves lightweight audit data while removing large intermediate artifacts.

### Failure Handling

If copying `4d.mp4`, writing bundled JSON, or cleanup fails after at least one Wan target was exported:

- do not silently ignore the error
- record the failure in `sample_issue_ledger.json` as `sample_failed`
- keep whatever files still exist in the sample working directory

Cleanup should run last so packaging artifacts are created before any deletion begins.

## Code Touch Points

### `scripts/wan_sample_export.py`

- always build and write aggregated `pose_meta_sequence.json`
- always build and write aggregated `smpl_sequence.json`
- continue writing optional per-frame `pose_meta_json/`
- expose enough finalized target metadata for the refined pipeline to know which target directories were written

### `scripts/offline_app_refined.py`

- after `run_refined_4d_generation()` and successful Wan export finalization, copy the final 4D mp4 into each exported target directory
- add a cleanup helper for sample working-directory intermediates
- guard cleanup behind:
  - successful refined completion
  - Wan export enabled
  - at least one exported target
  - `cleanup_sample_workdir_after_export = True`

### `scripts/wan_sample_types.py`

- add typed config fields for:
  - `copy_rendered_4d_to_targets`
  - `cleanup_sample_workdir_after_export`

### Config Files

- surface the new config fields in refined config variants so the behavior is explicit and adjustable

## Tests

Add tests for:

- aggregated `pose_meta_sequence.json` is written even when per-frame pose JSON is disabled
- aggregated `smpl_sequence.json` is written with frame records and JSON-safe payloads
- successful export copies the final 4D video into each target directory as `4d.mp4`
- cleanup removes heavy sample intermediates but preserves `sample_runtime.json` and `debug_metrics/`
- cleanup does not run when no Wan targets were exported

## Risks

The main risk is deleting artifacts too aggressively and making debugging harder. The design mitigates this by:

- only cleaning after successful Wan export with at least one target
- preserving `sample_runtime.json`
- preserving `debug_metrics/`
- adding bundled `pose_meta_sequence.json` and `smpl_sequence.json` into each target package

## Non-Goals

This design does not:

- remove the sample working directory itself
- replace existing target media outputs
- deduplicate copied `4d.mp4` across targets
- change the semantics of Wan face filtering or sample-level face-presence skipping
